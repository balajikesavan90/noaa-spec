# Evidence Matrix

This matrix maps the core NOAA-Spec reviewer-facing claims to exact repository evidence. It is intended to make the JOSS evidence boundary auditable without expanding the paper.

Evidence strength is labeled conservatively:

- `upstream-traceable fixture-backed`: the evidence has a committed raw/expected output pair plus recorded NOAA/NCEI source URL, retrieval date, and upstream checksum.
- `fixture-backed`: the evidence has committed raw/expected output pairs verified by `reproducibility/checksums.sha256`, but does not necessarily replay upstream NOAA retrieval.
- `unit-test-backed`: the behavior is covered by regression tests, but the small fixtures do not demonstrate the full supported surface.
- `documentation-only`: the item is a scope or boundary statement, not an executable evidence claim by itself.

The reproducibility boundary is `clean(committed_input) = committed_output` for the public `noaa-spec clean` CLI. Three small fixtures add upstream NOAA URL/date/checksum traceability: `real_provenance_example/`, `traceable_peru_il_2014_aa1_qc/`, and `traceable_albion_ne_2014_calm_aa1/`. The static curated example appendix in `artifacts/curated_examples/` is optional reviewer context selected from a committed candidate pool; it is not part of the checksum-backed reproducibility contract.

Core reviewed field-family coverage means source/control columns plus the mandatory NOAA families central to the public `noaa-spec clean` claim: `WND`, `CIG`, `VIS`, `TMP`, `DEW`, and `SLP`. Extended implemented coverage means additional NOAA families that are useful to users and remain documented, but are secondary evidence: they are implemented and unit-tested, with selected fixture evidence but not equal upstream-traceable real-data support for every family.

## Honest Gaps

- Three small fixtures are upstream-traceable to NOAA URLs, retrieval dates, and upstream checksums; this is still selected evidence, not a NOAA-wide upstream replay.
- The curated example appendix records NOAA station-year source URLs and line numbers, but only two curated examples have been promoted into committed upstream-traceable reproducibility fixtures.
- The static candidate pool retained for the curated appendix covers `TMP`, `VIS`, `WND`, and `AA1`; other supported families rely on tracked fixtures where present and unit tests rather than this optional appendix artifact.
- NOAA-Spec does not claim exhaustive coverage for all NOAA ISD / Global Hourly identifiers or every possible value within supported families.
