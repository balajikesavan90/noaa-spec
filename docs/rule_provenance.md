# Representative Rule Provenance

NOAA-Spec implements encoded parsing and normalization rules derived from the official NOAA ISD / Global Hourly documentation checked into `spec_sources/`. This page is reviewer-oriented traceability for representative rules. It is not a claim that every implemented rule is generated from, or linked to, a formal one-rule-to-one-provenance artifact.

For detailed source text, inspect the checked-in NOAA documentation under `spec_sources/isd-format-document-parts/`. For implementation details, inspect `src/noaa_spec/constants.py` for field rules and naming maps, and `src/noaa_spec/cleaning.py` for expansion, sentinel normalization, and QC sidecar emission.

| Source field / identifier | What NOAA-Spec does | Implementation to inspect | Supporting checked-in source docs |
| --- | --- | --- | --- |
| `TMP` air temperature | Splits value and quality code, scales tenths of degrees C, converts sentinel `9999` to null, emits `temperature_c`, `temperature_quality_code`, and `TMP__qc_*`. | `FIELD_RULES["TMP"]`, `FRIENDLY_COLUMN_MAP`, `_expand_parsed()` in `src/noaa_spec/cleaning.py` | `spec_sources/isd-format-document-parts/part-03-mandatory-data-section.md` |
| `DEW` dew point temperature | Applies the same composite value/quality structure for dew point, including sentinel-to-null and QC sidecars. | `FIELD_RULES["DEW"]`, `FRIENDLY_COLUMN_MAP` | `spec_sources/isd-format-document-parts/part-03-mandatory-data-section.md` |
| `SLP` sea-level pressure | Scales pressure to hPa, preserves pressure quality code, and treats documented missing sentinel as null. | `FIELD_RULES["SLP"]`, `FRIENDLY_COLUMN_MAP` | `spec_sources/isd-format-document-parts/part-03-mandatory-data-section.md` |
| `VIS` visibility | Splits distance, distance quality, variability code, and variability quality; emits `visibility_m` and related quality columns. | `FIELD_RULES["VIS"]`, `FRIENDLY_COLUMN_MAP` | `spec_sources/isd-format-document-parts/part-03-mandatory-data-section.md` |
| `WND` wind observation | Splits direction, direction quality, type code, speed, and speed quality; scales speed to m/s and preserves quality columns. | `FIELD_RULES["WND"]`, `FRIENDLY_COLUMN_MAP`, calm-wind handling in `src/noaa_spec/cleaning.py` | `spec_sources/isd-format-document-parts/part-03-mandatory-data-section.md` |
| Mandatory quality-code fields | Keeps NOAA quality codes as explicit columns instead of dropping them during measurement normalization. | `QUALITY_FLAGS`, `MANDATORY_QUALITY_FLAGS`, per-field `allowed_quality` entries | `spec_sources/isd-format-document-parts/part-03-mandatory-data-section.md` and `part-04-additional-data-section.md` |
| Numeric sentinels such as `9999`, `99999`, `999999` | Converts documented missing numeric sentinels to null measurement values and emits `SENTINEL_MISSING` in QC reason sidecars when applicable. | `FieldPartRule.missing_values`, `_is_missing_value()`, `_compute_qc_signals()` in `src/noaa_spec/cleaning.py` | Relevant field definitions in `part-03-mandatory-data-section.md` and `part-04-additional-data-section.md` |
| `AA1`-`AA4` liquid precipitation | Parses repeated precipitation period, amount, condition, and quality fields; preserves repeat index in names such as `precip_amount_1`. | `FIELD_RULES["AA"]`, `_REPEATED_IDENTIFIER_RANGES`, `_FRIENDLY_PATTERNS` | `spec_sources/isd-format-document-parts/part-04-additional-data-section.md` |
| `GA1`-`GA6` cloud layers | Parses repeated cloud-layer coverage, base height, type, and related quality-code fields. | `FIELD_RULES["GA"]`, `_REPEATED_IDENTIFIER_RANGES`, `_FRIENDLY_PATTERNS` | `spec_sources/isd-format-document-parts/part-15-cloud-and-solar-data.md` |
| `KA1`-`KA4` extreme temperature | Parses repeated extreme-temperature period, type, value, and quality fields; scales temperature values to degrees C. | `FIELD_RULES["KA"]`, `_REPEATED_IDENTIFIER_RANGES`, `_FRIENDLY_PATTERNS` | `spec_sources/isd-format-document-parts/part-24-temperature-data.md` |
| `MW1`-`MW7` present weather | Parses repeated present-weather codes and quality fields when present. | `FIELD_RULES["MW"]`, `_REPEATED_IDENTIFIER_RANGES`, weather code tables in `constants.py` | `spec_sources/isd-format-document-parts/part-28-weather-occurrence-data-extended.md` |
| Identifier and repeat validation | Accepts known NOAA identifiers, validates repeated family suffixes such as `AA1` or `GA2`, and rejects malformed repeated forms such as two-digit suffixes where unsupported. | `KNOWN_IDENTIFIERS`, `_REPEATED_IDENTIFIER_RANGES`, `is_valid_repeated_identifier()`, `get_field_rule()` | `spec_sources/isd-format-document-parts/part-04-additional-data-section.md` |
| Row-level usability summaries | Counts emitted `__qc_pass` columns to provide row-level interpretability fields; this is an engineering summary over parsed QC signals, not a NOAA source field. | `QC_STATUS_VALUES`, `QC_REASON_ENUM`, `USABILITY_METRIC_INDICATORS`, row summary logic in `src/noaa_spec/cleaning.py` | Implementation-defined; supporting NOAA field semantics come from the parsed source-field docs above |

The most useful reviewer spot-check path is to compare a fixture row in `reproducibility/minimal/station_raw.csv` with `reproducibility/minimal/station_cleaned_expected.csv`, then inspect the corresponding field rule in `src/noaa_spec/constants.py`.
